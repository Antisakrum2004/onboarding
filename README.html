01 Общая информация о проекте

Что это

Онбординг 1С АйтиЛаб (он же «Welcome Book») — интерактивная HTML-страница, которая знакомит новых сотрудников компании с процессами, инструментами, командой и правилами работы. Это Single-Page Application (SPA), реализованная в одном HTML-файле без внешних зависимостей (кроме Google Fonts).

Формат
Один HTML-файл (~98 КБ), содержит CSS, JS и весь контент
Фреймворк
Без фреймворков — чистый HTML/CSS/JS (vanilla)
Содержание
4 части, 14 разделов, 20 вопросов финального теста
Хостинг
GitHub Pages — автоматически из ветки main
Репозиторий
github.com/Antisakrum2004/onboarding
URL
antisakrum2004.github.io/onboarding/

Почему один файл?

Такая архитектура выбрана целенаправленно. Онбординг должен быть максимально простым в деплое: один файл = одна загрузка, нет сборки, нет зависимостей, нет серверной части. Любой человек может открыть файл в любом текстовом редакторе, найти нужную функцию и поправить текст. Это критично, потому что контент будет часто меняться.

Принцип рендеринга

Приложение работает по принципу «состояние → рендер». При каждом изменении state вызывается render(), который полностью перестраивает DOM:

// Главный цикл:
function render() {
  renderSidebar();     // Перестроить навигацию
  renderContent();    // Перестроить центральную область
  renderBottomAction(); // Перестроить нижнюю панель
}

Все HTML-строки генерируются через конкатенацию в JavaScript. Контент каждой из 14 секций — это отдельная функция (s1()...s14()), которая возвращает HTML-строку с готовой разметкой.

04 Система навигации и маршрутизация

Приложение не использует URL-маршрутизацию (no hash, no history API). Вместо этого — внутренняя маршрутизация через state.currentView:

function navigate(view, sectionIndex) {
  if (view === 'welcome')  { state.currentView = 'welcome'; }
  if (view === 'section')  { state.currentView = 'section'; state.currentSection = sectionIndex; }
  if (view === 'test')     { state.currentView = 'test'; }      // только если все 14 разделов пройдены
  if (view === 'results')  { state.currentView = 'results'; }
  if (view === 'admin')    { state.currentView = 'admin'; }
  saveState(); render(); window.scrollTo(0, 0);
}
ViewЧто рендеритсяКогда доступен
welcomeПриветственная страница с кнопкой «Начать»Всегда (кнопка 🏠 в сайдбаре)
sectionОдин из 14 разделов (s1...s14)Всегда (навигация свободная)
testФинальный тест (20 вопросов)Только если completed.length >= 14
resultsРезультаты теста с конфеттиПосле завершения теста
adminМодальное окно редактированияВсегда (кнопка ⚙️ в сайдбаре)

Сайдбар

Сайдбар генерируется динамически через renderSidebar(). Структура:

  • 🏠 Главная — навигация на welcome (класс .nav-home)
  • Прогресс-бар — «X из 14» с жёлтой полоской
  • 4 группы разделов — ЧАСТЬ 1...4 с заголовками (класс .nav-group-title)
  • 14 пунктов — с emoji, чекмарком, active/completed стейтами
  • Финальный тест — заблокирован (серый) / разблокирован (жёлтый) / пройден (зелёный)
  • ⚙️ Редактировать контент — открывает admin-панель

Мобильная навигация

На экранах <900px сайдбар скрывается и заменяется мобильным header с бургер-меню. Мобильное меню — выдвижная панель слева с overlay. Навигация идентична десктопной.

05 Управление состоянием (State)

Глобальный объект state

let state = {
  currentView: 'welcome',   // Какой экран показан
  currentSection: 0,          // Индекс текущего раздела (0-13)
  completed: [],               // Массив ID пройденных разделов ['s1','s2',...]
  testAnswers: {},             // {индекс_вопроса: выбранный_ответ}
  testStarted: false,          // Тест запущен
  testFinished: false,         // Тест завершён
  testScore: 0,                // Количество правильных ответов
  testTime: 0,                 // Время прохождения (мс)
  testStartTime: null,         // Timestamp старта
  testCurrentQ: 0,             // Текущий вопрос (0-19)
  checklistState: {},          // {'d0_0': true, 'd1_2': false, ...}
};

State полностью сериализуется в localStorage при каждом изменении. Это позволяет сохранить прогресс между сессиями браузера. При загрузке страницы loadState() восстанавливает state из localStorage через Object.assign() — новые поля (добавленные в обновлениях) получают дефолтные значения.

07 Система тестирования (20 вопросов)

Структура вопроса

{ q: 'Текст вопроса', opts: ['Вариант А', 'Вариант Б', 'Вариант В'], ans: 1 }
// ans — индекс правильного ответа (0, 1 или 2)

Логика теста

  • Тест доступен только после прочтения всех 14 разделов (completed.length >= 14)
  • При старте запускается таймер, ответы обнуляются
  • Навигация: назад/вперёд по вопросам, ответ обязателен для перехода
  • Прогресс-бар показывает все 20 вопросов (серый → жёлтый → зелёный/красный)
  • По завершении — подсчёт: минимум 16/20 (80%) для прохождения
  • При провале — можно повторить (retryTest) или сбросить всё (resetAll)
  • При успехе — конфетти 🎉 + кнопка «Начать заново»

08 Admin-панель (редактирование контента)

Admin-панель встроена прямо в страницу — кнопка ⚙️ Редактировать контент внизу сайдбара. Открывается как модальное окно поверх контента.

Три вкладки:

ВкладкаФункциональность
📖 РазделыСписок всех 14 разделов. Клик → открывается textarea с HTML-контентом. Можно редактировать, сохранить, посмотреть оригинал. Изменённые разделы помечаются зелёным бейджем.
🧠 ТестСписок всех 20 вопросов. Клик → три поля: текст вопроса, варианты ответов (по одному на строку), номер правильного ответа. Можно сохранить или удалить кастом.
📝 ИсторияЛог всех изменений с датой и описанием (максимум 50 записей).

Кнопки действий:

  • 📥 Экспорт JSON — скачивает файл со всеми кастомными изменениями
  • 📤 Импорт JSON — загружает JSON-файл с изменениями поверх текущих
  • 🗑 Сбросить изменения (N) — удаляет все кастомные правки, возвращает стандартный контент
  • 🔄 Сбросить прогресс — сбрасывает прохождение (не трогает кастомный контент)
  • ✕ Закрыть — закрывает панель, возвращает на главную

Как это работает технически

Кастомный контент хранится в localStorage под ключом onboarding-aytilab-custom. Это объект {'s1': 'HTML...', 'q3': {q:'...',opts:[...],ans:1}, ...}.

В конце JavaScript-кода стоит перехват:

var _originalGetSectionContent = getSectionContent;
getSectionContent = function(idx) {
  var custom = getCustomContent();
  if (custom['s' + (idx+1)]) return custom['s' + (idx+1)];
  return _originalGetSectionContent(idx);
};

То есть: если для раздела есть кастомный контент в localStorage — он подставляется вместо стандартного. Если кастома нет — используется функция s1()...s14().

💡Workflow для совместной работы: ответственный редактирует через admin-панель → нажимает «Экспорт JSON» → сохраняет файл → отправляет разработчику → разработчик переносит изменения прямо в код функций s1()...s14() → пушит на GitHub → все пользователи видят обновление.

13 Известные ограничения и особенности

  • Нет серверной части — всё работает в браузере. Нельзя собрать аналитику «кто прошёл онбординг» без сторонних сервисов.
  • localStorage ≈ 5-10 МБ — если контент станет очень большим, могут быть проблемы. На практике для текстового онбординга этого более чем достаточно.
  • Нет версионирования контента — если стандартный контент в коде изменится, а у пользователя старый localStorage с кастомом — кастом останется поверх. Сброс решает проблему.
  • Все в одном файле — при ~1000 строк это нормально, но если контент будет расти (30+ разделов), стоит рассмотреть вынос данных в отдельный JSON-файл и подгрузку через fetch.
  • Тест доступен только после всех 14 разделов — это сделано намеренно, чтобы сотрудник точно прочитал весь материал. Можно изменить порог в функции navigate().
  • Кастомные изменения через admin — localStorage — то есть каждый пользователь видит только свои правки. Для глобальных изменений нужно пушить в код. Это оговорено в UI admin-панели.
  • Unicode эмодзи — используются escape-последовательности вида \u{1F3E2} в JS. Это работает во всех современных браузерах, но при редактировании кода нужно быть аккуратным — не заменять их на символы, иначе можно сломать кодировку.

14 Гайд по внесению изменений

🔧 Как изменить текст раздела

Найти функцию s1()...s14() (поиск по названию раздела или номеру). Текст — это HTML-строка, возвращаемая функцией. Можно использовать любые HTML-теги из списка компонентов (info-card, value-grid, timeline и т.д.).

// Пример — изменить раздел «О компании» (s1):
function s1() {
  return '<h3>Новый заголовок</h3>'
    + '<p>Новый текст</p>';
}

➕ Как добавить новый раздел

// 1. Добавить в массив SECTIONS (после s14):
{id:'s15', num:15, part:4, emoji:'\u{1F4C4}', title:'Новый раздел', short:'Новое'}

// 2. Обновить прогресс: 14 → 15 везде (renderSidebar, navigate)
//    - "0 из 14" → "0 из 15"
//    - "completed.length<14" → "completed.length<15"

// 3. Создать функцию:
function s15() { return '<h3>Контент</h3><p>...</p>'; }

// 4. Добавить case в getSectionContent():
case 14: return s15();

// 5. Закоммитить и пушить

❓ Как добавить/изменить вопрос теста

// В массиве QUESTIONS:
{
  q: 'Текст вопроса?',
  opts: ['Вариант А', 'Вариант Б', 'Вариант В'],
  ans: 1  // Индекс правильного (0 = первый, 1 = второй, 2 = третий)
}

// Также обновить: "20 вопросов" → "21 вопрос" в UI

🎨 Как изменить цветовую тему

// Всё в блоке :root в начале CSS:
:root {
  --yellow: #F5C327;     // Меняем основной акцент
  --black: #1A1A1A;      // Меняем цвет текста
  --page-bg: #F4F4F2;    // Меняем фон
  // ... и так далее
}

🧹 Как сбросить данные пользователя

В консоли браузера (F12 → Console):

localStorage.removeItem('onboarding-aytilab-v2');
localStorage.removeItem('onboarding-aytilab-custom');
localStorage.removeItem('onboarding-aytilab-changelog');
location.reload();
